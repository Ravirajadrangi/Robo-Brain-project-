In order to make ModalDB accessible, execute the following steps:

Step 1: Run the configuration script, ModalDB/configure.sh: (This puts ModalDB in your PYTHONPATH envvar)

    cd /path/to/ModalDB
    source ./configure.sh

Step 2: start MongoDB in the appropriate location: (For this demo, make it ModalDB/demos/data/db

    cd /path/to/ModalDB/demos
    mongod --dbpath=./data/db

II. Basic Principles of ModalDB¶

ModalDB is a database optimized for applications that need to efficiently interact with data of multiple 'modes,' or formats, such as images, text and audio. It lazily loads all data that is stored on disk, while providing a interface to users that abstracts away the details of how data is retrieved. It also provides the ability to create a flexible, hierarchical structure to your database.

The core concepts that developers should familiarize themselves with are DataObjects and ModalSchemas. DataObjects are essentially data containers - they group related chunks of data ("items") together and provide an interface to interact with them. Users specify which DataObjects contain what forms of data and how it should be efficiently accessed using a ModalSchema.

These are perhaps best explained with an example:

Schema Example:¶

In [2]: 
from ModalDB import Frame, Video # Both subclasses of ModalDB.DataObjec
my_schema = ModalSchema({
                            # Frames consist of an image (stored on disk) and accompanying subtitles (stored in memory)
                            Frame:  {
                                        'image':{
                                                    'mode':'disk',                   # store it on disk, instead of mongodb
                                                    'filename':'image.jpg',          # filename to store data under
                                                    'load_func':lambda p: imread(p), # how to load from disk. (p = path to file)
                                                    'save_func':lambda x, p: imsave(p, x) # how to save to disk, when set. (optional)
                                                },
                                        'subtitles':{
                                                    'mode':'memory'
                                                    }
                                    },

                            # Videos contain subtitles, which we can store in memory (mongodb) as strings.
                            Video:  {
                                        'thumbnail':{
                                                        'mode':'disk',
                                                        'filename':'thumbnail.png',
                                                        'load_func':lambda p: imread(p),
                                                        'save_func':lambda x, p: imsave(p, x)
                                                    },
                                        
                                        # Videos also contain collections of 'Frame' dataobjects (defined below)
                                        'contains':[Frame]
                                    }
                        })

Note that both Frame and Video are subclasses of DataObject. (These are defined in the ModalDB source code, as they will be used quite frequently.) This ModalSchema specifies that Frames will contain both an image and subtitles. Furthermore, it specifies that Videos contain a thumbnail (an image) and a set of Frames.

III. Creating and Populating a ModalDB¶

Creation¶

Now that we have defined a schema for our database, let's construct one, insert some items, and start to use it.

import os
data_dir = os.path.join(os.getcwd(), './data/') # we will construct our database in the 'demos' directory
modaldb_client = ModalClient(root=data_dir, schema=my_schema) # here, we specify the DB's location (on disk) and schema.
modaldb_client.clear_db() # empty the database just in case.

Basic Insertion¶

Let's insert a video into the database. We can do this using the ModalClient's method insert. This takes in a specification of what type of DataObject you are inserting, what to call it, as well as a dict mapping individual item names to their location on disk (if it is to be stored on disk) or their values (if it is to be stored in memory). item_dict = {'thumbnail':'./raw_data/thumbnail.jpg'} # our video's 'thumbnail' item is currently located at ./raw_data/thumbnail.jpg
my_video = modaldb_client.insert(Video, 'my_video', item_dict, method='cp') # method='cp' specifies that we will *copy*, not move, all disk data.

Now let's add a couple of Frame objects to the my_video

IV. Accessing and Manipulating DataObjects¶

Now that we have added DataObjects to the database, let's play around with them.

Retrieving DataObjects¶

We can retrieve DataObjects using eModalClient's get and get_random methods, as well as DataObject's get_child and get_random_child methods. These are demonstrated below:

In [8]: 
# To retrieve DataObjects that don't have a parent
video = modaldb_client.get(Video, 'my_video')

# These work for DataObjects that do have parents
frame = video.get_child(Frame, 'frame1')
frame = video.get_child('frame1')        # don't have to specify child's type, since there is only one.
frame = video.get_random_child(Frame)
frame = video.get_random_child()         # again, don't have to specify child's type since there is only one.
